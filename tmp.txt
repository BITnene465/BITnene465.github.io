【五、数据模型要求（TS interface）】

请在实现 data 和 lib/mdx.ts 时使用以下数据结构（可以按需要稍微拓展，但不要减少字段）：

2. 项目数据 `Project`（data/projects.ts）：

```ts
  id: string
  name: string
  period?: string           // '2023–2025' 等
  role?: string             // 'Lead', 'Contributor'
  description: string
  tags?: string[]
  links?: {
    github?: string
    demo?: string
  }
}
```

3. 个人信息 `Profile`（data/profile.ts）：

```ts
export interface Profile {
  name: string
  tagline: string
  bio: string
  location?: string
  currentPosition?: string
  researchAreas: string[]
  contacts: {
    email?: string
    github?: string
    scholar?: string
    twitter?: string
    [key: string]: string | undefined
  }
  values: { title: string; description: string }[]
  timeline: {
    year: string
    title: string
    description?: string
  }[]
}
```

4. Blog 文章 meta `PostMeta`（lib/mdx.ts）：

```ts
export type PostType = 'article' | 'note' | 'link'

export interface PostMeta {
  slug: string
  title: string
  date: string              // ISO 字符串
  tags: string[]
  type: PostType
  summary?: string
  externalUrl?: string      // type === 'link' 时必填
  component: React.ComponentType
}
```

MDX 文件 frontmatter 统一格式：

```mdx
---
title: "标题"
date: "2025-11-17"
tags: ["research", "note"]
type: "article"       # "article" / "note" / "link"
summary: "一两句摘要"
externalUrl: ""       # 仅当 type 为 "link" 时必填
---

正文内容……
```

【六、UI 与动画规范】

1. 基本视觉：

   * 整体偏简洁科技感，主色可以使用蓝紫 + 辅色青绿或淡粉，背景可采用深色或中性的浅灰。
   * 二次元/游戏元素主要出现在：

     * Home 的 Hero 插画区域；
     * Playground 页面；
     * 少量 icon 或背景装饰。

2. Tailwind 使用：

   * 在 tailwind.config 中定义基础色 token，如 `bg`, `bg-alt`, `primary`, `primary-soft`, `accent`, `text-main`。
   * 页面布局常用：

     * 外边距：手机端 `px-4`，桌面 `px-8` 以上；
     * 内容最大宽度：`max-w-5xl` 或 `max-w-6xl` 居中。

3. 通用组件：

   * Button：

     * 至少支持 `variant="primary"` 和 `variant="ghost"`。
   * Card：

     * 有圆角、浅阴影和内边距；
     * hover 使用 Framer Motion 实现轻微上浮 + 放大 + 阴影增强。
   * Tag：

     * 小圆角标签，用于展示论文/文章/技术标签。
   * Section：

     * 用于统一每个页面区块的标题 + 描述 + 内容间距。

4. 动画（lib/motion.ts）：

   * 统一定义：

     * `pageTransition`：页面级淡入淡出 + 轻微上下位移。
     * `cardHover`：卡片 hover 缩放 + 上移。
   * 页面组件使用 `pageTransition`；
   * 卡片组件使用 `cardHover`；
   * 不要自作主张添加过于复杂、旋转、闪烁的动画，也不要做大面积持续运动的背景。
